# Architecture

Living docs for the Briefed system. Anything here that a running service
can regenerate from metadata **must** be regenerated by `make docs` — never
hand-maintained — to prevent drift.

## Contents

- `data-model.md` — Mermaid `erDiagram` of the Postgres schema. Hand-authored
  and updated when Alembic migrations land. Phase 0 ships the placeholder;
  Phases 1–4 populate tables as they're defined.
- `pipeline.md` — ingest → classify → summarize → digest flow; written in
  Phase 2 when the pipeline becomes concrete.
- `threat-model.md` — see `docs/security/`.

Generated artifacts are listed in `docs/architecture/README.md` once they
exist; the drift check in CI lives in `.github/workflows/ci.yml`.
