# packages/config

Seed defaults, sample policies, and JSON Schemas derived from Pydantic
settings models. The backend ships the same Pydantic classes; this package
publishes them as machine-readable schemas so operators and the frontend
can validate config without a running FastAPI process.

## Layout

```
config/
├── seeds/
│   ├── rubric.default.json        # starter priority rubric (Phase 2 seeds it)
│   └── known_waste_senders.json   # starter unsubscribe block-list
├── samples/
│   └── user_preferences.sample.json
└── schemas/
    ├── user_preferences.schema.json
    ├── connected_account.schema.json
    └── rubric_rule.schema.json
```

Phase 0 ships the directory skeleton + a `.gitkeep` in each subdir. Schemas
are generated by `backend/scripts/dump_config_schema.py` (lands in Phase 2);
seeds are hand-authored and reviewed by the product owner.
